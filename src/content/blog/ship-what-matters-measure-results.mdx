# Ship What Matters, Measure Results

Draft scaffold — outline subject to change.

## Premise

- Most teams ship output, not outcomes.
- The path to durable impact is thin slices, measurable goals, and honest reviews.
- Strategy is choosing what not to do; measurement is validating that choice.

## Operating Model

- Define the smallest slice that creates user or business value.
- Set a measurable goal that can move in days, not quarters.
- Instrument the slice before shipping it.
- Review outcomes weekly; kill or double down based on evidence.

## Planning Cadence (lightweight)

- 6-week objective (1–2 goals) with explicit non-goals.
- Weekly review (30–45 minutes) focused on:
  - What moved and why?
  - What did we learn?
  - What do we stop, start, or continue?
- End-of-cycle write-up: 1–2 pages, data + narrative.

## Thin Slices

- “What is the smallest thing we can ship that delivers value and can be measured within a week?”
- Prefer slicing by user journey or “job to be done,” not by layers.
- Build only the guardrails needed to ship safely.

## Measurement Loops

- Leading indicators (fast feedback): click-through, latency, task completion time.
- Lagging indicators (business impact): retention, conversion, revenue, cost.
- Define success and stopping conditions upfront.

## Example Metrics (pick few)

- Conversion: visitor → signup → activated → retained.
- Experience: TTFB, p95 latency, error rate, task completion time.
- Efficiency: cost per request/tenant/feature, build time, deploy time.

## Anti-patterns

- Shipping features without a hypothesis or metric.
- Measuring everything and learning nothing (no decisions).
- Big-bang projects with no interim value checkpoints.
- Post-hoc goal setting (“we shipped it, so it worked”).

## Templates (to add)

- One-page spec:
  - Problem, hypothesis, metric(s), slice plan, risks, de-scope plan.
- Review doc:
  - Data snapshot, what we learned, what we’ll do next (or stop).
- Dashboard:
  - 3–5 tiles max: leading + lagging signals.

## Checklist

- [ ] Clear problem statement and hypothesis.
- [ ] Single measurable goal with target and timeframe.
- [ ] Instrumentation plan (metrics/logs/traces) in place before shipping.
- [ ] Thin slice scoped to a week or less.
- [ ] Weekly review scheduled with a decision rubric.
- [ ] End-of-cycle write-up and roll-forward plan.
